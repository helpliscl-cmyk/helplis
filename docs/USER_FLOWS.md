# User Flows

Flujos implementados:

1. Registro local: `/register`.
2. Login local: `/login`.
3. Activación: `/activate` o `/activate/[publicCode]`, con publicCode, activationCode, usuario, perfil, contacto y privacidad inicial.
4. Dashboard: resumen, dispositivos, perfiles, contactos, escaneos, ubicaciones, privacidad, cuenta y soporte.
5. Marcar dispositivo como perdido, encontrado o reactivado.
6. Revisar ficha pública desde dashboard.

El flujo permite activar con alias/nombre visible, un contacto y privacidad básica sin obligar información médica.
